# 📡 Radar

CLI-утилита на Go для диагностики работы раздельного маршрутирования (**Split Tunneling**) в режимах TUN (Hysteria, sing-box, dae, mihomo).

Программа одновременно проверяет внешние IP-адреса через разные доменные зоны и тестирует доступность заблокированных ресурсов, выявляя, уходит ли трафик через прокси или напрямую.

## ✨ Особенности

- **Параллельный запуск**: Опрос IP-резолверов и чекеров сайтов происходит конкурентно в горутинах.
- **Детекция Split Tunneling**: Автоматическое сравнение полученных IP-адресов. Если обнаружены отличия, утилита подтверждает работу разделения трафика.
- **Проверка блокировок**: Тестирование доступности популярных сервисов (`Instagram`, `X.com`, `OpenAI` и др.) с жестким таймаутом в 3 секунды.
- **Изолированный сетевой стек**: HTTP-клиент игнорирует системные переменные окружения (`http_proxy`), тестируя исключительно таблицу маршрутизации TUN-интерфейса.
- **Кросс-компиляция**: Готовый `Makefile` для сборки под Linux, Windows и macOS.

## 🚀 Использование

### Быстрый запуск
```bash
go run main.go
```

### Режим отладки (Debug)
Для вывода подробных логов сравнения IP-адресов используйте флаг `-debug`:
```bash
go run main.go -debug
```

### Режим без прокси
По умолчанию приложение ищет прокси на хосте и если находит - использует.
Данный флаг '-no-proxy' игнорирует активный прокси.
```bash 
go main.go -no-proxy
```

## 🛠️ Сборка (Makefile)

Проект автоматизирован с помощью `Makefile`. Сборка оптимизирована — флаги компиляции `-ldflags="-s -w"` уменьшают размер бинарников на ~40%.

- **Собрать под все ОС сразу**:
  ```bash
  make
  ```
- **Собрать только под конкретную платформу**:
  ```bash
  make windows      # build/radar-windows-amd64.exe
  make linux        # build/radar-linux-amd64
  make macos-intel  # build/radar-darwin-amd64
  make macos-arm    # build/radar-darwin-arm64
  ```
- **Очистить папку сборки**:
  ```bash
  make clean
  ```
